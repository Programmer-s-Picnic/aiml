1. Overview

Let us start from the most basic level and build up to a practical and premium understanding.

Why it exists

Real data is messy

In real life, data points are usually not perfectly aligned. We often have observations like study hours vs marks, ad spend vs sales, or temperature vs electricity usage.

We want a curve that captures the general trend. That is where polyfit() becomes useful.

What it returns

It returns coefficients

np.polyfit() does not directly return predicted values. It returns the coefficients of the polynomial that best fits the data.

Example: for degree 1, it returns slope and intercept.

Main use

Trend fitting

It is widely used for line fitting, curve fitting, approximation, and as a stepping stone toward regression ideas used in machine learning.

2. Theory and Intuition

This section explains what is happening behind the scenes in a simple and practical way.

Best-fit line

For degree 1

y = mx + b

Here, m is the slope and b is the intercept. When you use np.polyfit(x, y, 1), NumPy finds the best values of m and b.

Best-fit curve

For degree 2

y = ax² + bx + c

When the data bends like a curve, a quadratic fit may be better. Then np.polyfit(x, y, 2) returns a, b, and c.

Important intuition: polyfit() is trying to reduce the total error between actual values and predicted values. In simple words, it tries to place the line or curve so that it matches the overall trend as well as possible.
Least squares idea

What does “best fit” really mean?

Suppose the actual data values are y and the fitted values are ŷ. Then the model wants the total squared error to be as small as possible.

Minimize: Σ (y - ŷ)²

Squaring the errors makes negative and positive errors both count, and large mistakes are punished more strongly.

3. Syntax

The syntax is short, but the idea behind it is very powerful.

np.polyfit(x, y, degree)
Part Meaning Example
x Input values or independent variable Study hours
y Output values or dependent variable Marks
degree Polynomial degree 1 for line, 2 for parabola, 3 for cubic
degree 1 = line degree 2 = curve degree 3 = more flexible

4. Step-by-Step Examples

We will build understanding one level at a time.

Example 1 — Straight line fitting

Let us take a small dataset and fit a straight line to it using degree 1. 

import numpy as np

x = np.array([1, 2, 3, 4, 5])
y = np.array([2, 4, 5, 4, 5])

coeff = np.polyfit(x, y, 1)

m = coeff[0]
b = coeff[1]

print("Slope =", m)
print("Intercept =", b)
Meaning: this returns two values because a line has two coefficients: slope and intercept.

Step 1

x stores input values and y stores output values.

Step 2

np.polyfit(x, y, 1) means fit a polynomial of degree 1, that is, a straight line.

Step 3

The result contains m and b for the equation y = mx + b.

Example 2 — Predicting values using the line

Once you get the coefficients, you can compute predicted values. 

import numpy as np

x = np.array([1, 2, 3, 4, 5])
y = np.array([2, 4, 5, 4, 5])

m, b = np.polyfit(x, y, 1)

y_pred = m * x + b

print("Predicted values:", y_pred)

Here we use the fitted line equation directly to get predictions for the same x-values.

Example 3 — Quadratic fitting

Some data bends upward or downward. In that case, a curve may be better than a line. 

import numpy as np

x = np.array([1, 2, 3, 4, 5])
y = np.array([1, 4, 9, 16, 25])

coeff = np.polyfit(x, y, 2)

a = coeff[0]
b = coeff[1]
c = coeff[2]

print("a =", a)
print("b =", b)
print("c =", c)

The equation is:

y = ax² + bx + c

Degree 2 returns three coefficients because a quadratic equation has three parts.

Example 4 — Using poly1d for a cleaner workflow

np.poly1d() converts coefficients into a polynomial object, which is easier to use. 

import numpy as np

x = np.array([1, 2, 3, 4, 5])
y = np.array([1, 4, 9, 16, 25])

coeff = np.polyfit(x, y, 2)
p = np.poly1d(coeff)

print(p)
print("Prediction at x = 6:", p(6))

This is cleaner than manually writing a*x**2 + b*x + c every time.

Example 5 — Plotting the fitted result

Plotting makes understanding much easier. 

import numpy as np
import matplotlib.pyplot as plt

x = np.array([1, 2, 3, 4, 5])
y = np.array([2, 4, 5, 4, 5])

coeff = np.polyfit(x, y, 1)
p = np.poly1d(coeff)

x_new = np.linspace(x.min(), x.max(), 100)
y_new = p(x_new)

plt.scatter(x, y, label="Actual data")
plt.plot(x_new, y_new, label="Best-fit line")
plt.title("NumPy polyfit Example")
plt.xlabel("x")
plt.ylabel("y")
plt.legend()
plt.grid(True)
plt.show()
Why use linspace? It creates many x-values between the minimum and maximum, so the fitted line or curve looks smooth.

5. Degree Understanding Table

Choosing the right degree is important. Too low can miss the pattern. Too high can overfit.

Degree Shape Typical Equation Form When to Use
1 Straight line y = mx + b When the trend is almost linear
2 Parabolic curve y = ax² + bx + c When the data bends smoothly
3 Cubic curve y = ax³ + bx² + cx + d When the trend changes direction more than once
4+ More flexible Higher-order polynomial Use carefully, risk of overfitting

6. Common Mistakes

These are the issues learners face most often when using polyfit().

Mistake 1

Using the wrong degree

Problem:

If you force a straight line on strongly curved data, the fit will be poor.

Better:

Inspect the data first and choose a degree that matches the trend without becoming unnecessarily complex.

Mistake 2

Forgetting the coefficient order

Problem:

Learners often think the coefficients come in low-power-to-high-power order.

Better:

NumPy returns coefficients from highest power to constant term.

Mistake 3

Overfitting

Problem:

Choosing a very high degree can make the polynomial follow tiny random variations instead of the real trend.

Better:

Keep the degree as simple as possible while still capturing the pattern.

Mistake 4

Thinking polyfit gives a machine learning model object

Problem:

polyfit() returns coefficients, not a full scikit-learn model.

Better:

Use np.poly1d() if you want an easy callable polynomial object.

Very important: polyfit() is excellent for understanding fitting and regression ideas, but for larger machine learning workflows, people often move to tools like scikit-learn.

10. Mini Project — Student Marks Predictor

This mini project ties together fitting, prediction, and practical thinking.

Project idea

Study hours vs marks

Suppose you have data showing how many hours students studied and what marks they got. Use polyfit() to model the relationship and predict marks for a new study duration.

  • Create arrays for study hours and marks
  • Fit a degree 1 polynomial
  • Build a poly1d model
  • Predict marks for a new value
  • Plot the actual data and fitted line
import numpy as np
import matplotlib.pyplot as plt

hours = np.array([1, 2, 3, 4, 5, 6])
marks = np.array([38, 45, 57, 63, 74, 82])

coeff = np.polyfit(hours, marks, 1)
model = np.poly1d(coeff)

predicted_for_7 = model(7)

print("Model:", model)
print("Predicted marks for 7 hours:", predicted_for_7)

x_new = np.linspace(hours.min(), 7, 100)
y_new = model(x_new)

plt.scatter(hours, marks, label="Actual data")
plt.plot(x_new, y_new, label="Best-fit line")
plt.xlabel("Study hours")
plt.ylabel("Marks")
plt.title("Student Marks Predictor")
plt.legend()
plt.grid(True)
plt.show()
Premium insight: Even though this example is simple, it teaches the same thinking used in real predictive systems: take data, fit a mathematical relationship, then use it for estimation.
